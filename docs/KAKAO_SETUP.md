# 🚇 카카오 REST API 키 발급 가이드

입지 점수(주변 지하철·학교·마트·병원) 기능 활성화를 위해 필요합니다.

## 1단계 — 카카오 개발자 계정

1. https://developers.kakao.com 접속
2. 우측 상단 **[로그인]** → 카카오 계정으로 로그인
3. 처음이면 개발자 등록 동의 필요

## 2단계 — 애플리케이션 생성

1. 상단 메뉴 **[내 애플리케이션]** 클릭
2. **[애플리케이션 추가하기]** 버튼
3. 양식 입력:
   - 앱 이름: `부동산분석` (아무거나)
   - 사업자명: 본인 이름 또는 `개인`
   - 카테고리: `생활 > 부동산` 또는 기타 적절히
4. **[저장]**

## 3단계 — REST API 키 복사

1. 생성된 앱 클릭 → **[앱 키]** 페이지
2. 4가지 키 중 **REST API 키** 복사 (32자 정도)
   - 네이티브 앱 키 ❌ (모바일용)
   - JavaScript 키 ❌ (웹 프론트용)
   - **REST API 키** ✅ (서버용, 우리가 쓸 것)
   - Admin 키 ❌ (관리용)

## 4단계 — 🔥 카카오맵 서비스 활성화 (필수)

**이 단계 빠지면 403 NotAuthorizedError 발생합니다.**

1. 좌측 메뉴 **[제품 설정]** 또는 **[앱 설정 > 카카오맵]**
2. **카카오맵** 항목 → **활성화 토글 ON**
3. 저장

> 카카오는 앱별로 사용할 서비스를 명시적으로 켜야 합니다. 키 발급 = 즉시 사용 가능 아님.

## 5단계 — 플랫폼 등록 (선택, 보통 안 해도 됨)

REST API는 도메인 제한이 약해서 로컬 호출이 됩니다. 만약 안 되면:
- 좌측 메뉴 **[플랫폼]** → **[Web 플랫폼 등록]** → 사이트 도메인 `http://localhost:8501` 입력

## 6단계 — .env 파일에 추가

```powershell
notepad .env
```

이미 있는 `KAKAO_REST_API_KEY=` 뒤에 키 붙여넣기:

```
KAKAO_REST_API_KEY=AbCdEf123...
```

저장 후 Streamlit 재시작:

```powershell
Get-Process python | Where-Object { $_.Path -like "*\.venv\*" } | Stop-Process -Force
.\.venv\Scripts\python.exe -m streamlit run src\ui\streamlit_app.py
```

## 7단계 — 사용

1. 시드추천 탭 → 검색 실행
2. 결과 위에 **🚇 입지점수 계산** 체크박스 표시됨
3. 체크 시 상위 30개 단지에 대해 카카오 API 자동 호출 (캐시됨)
4. 결과 표에 다음 컬럼 추가:
   - **입지점수(카카오)** — 종합 점수 (0~100)
   - **주변지하철** / **주변학교** / **주변마트** / **주변병원** (1km 이내 개수)

## ⚠️ API 사용 제한 (무료 플랜)

| 항목 | 한도 |
|---|---|
| 일일 쿼터 | 100,000 호출 (개인 무료) |
| 초당 호출 | 약 10회 |
| 캐시 권장 | 한번 조회한 단지는 `data/processed/apt_locations.json` 에 저장됨 → 재호출 안 함 |

상위 30개 단지에 대해 4개 카테고리씩 → 1회 검색당 약 120 API 호출. 충분히 무료 한도 내.

## 점수 산식

```
점수 = min(100, 지하철×20 + 학교×5 + 마트×10 + 병원×3)
```

| 시나리오 | 점수 |
|---|---|
| 지하철역 도보 5분 거리(1개) + 학교 3곳 + 마트 2곳 | 20+15+20 = 55 |
| 지하철 환승역 인접(2개) + 학교 5곳 + 마트 3곳 + 병원 4곳 | 40+25+30+12 = 100 (포화) |
| 외곽 신축, 지하철 없음 | 0~15 |

## 점수 가중치 조정

`src/analysis/location.py` 의 `lookup_or_fetch` 함수 내 점수 산식 직접 수정 가능:

```python
score = min(100.0,
            n_subway * 20    # 지하철 가중치
            + n_school * 5   # 학교
            + n_mart * 10    # 마트
            + n_hospital * 3) # 병원
```

학군 중심으로 보고 싶으면 `n_school * 15` 등으로 변경.
